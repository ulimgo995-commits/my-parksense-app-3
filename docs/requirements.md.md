# 📘 [Requirements] ParkSense - 목적지 기반 주차장 혼잡도 안내 서비스

## 1. 프로젝트 개요

### 서비스명

ParkSense

### 서비스 소개

ParkSense는 운전자가 목적지 주변 공영주차장의 혼잡도를 미리 확인하여 가장 적합한 주차장을 선택할 수 있도록 도와주는 지도 기반 웹 서비스입니다.

현재 대부분의 지도 서비스는 주차장의 위치는 제공하지만 현재 얼마나 여유가 있는지 직관적으로 확인하기 어렵습니다.

ParkSense는 지도 위에서 혼잡도를 색상으로 표시하여 사용자가 가장 적합한 주차장을 빠르게 선택할 수 있도록 합니다.

---

# 2. 프로젝트 목표

이번 프로젝트는 메이커톤 MVP입니다.

MVP에서는 서울시 공영주차장 약 20~30개를 대상으로 서비스를 구현합니다.

실시간 공공데이터 API를 쉽게 연결할 수 있도록 구조를 설계하지만 메이커톤 기간 내 안정적인 시연을 위해 현재 혼잡도와 가능 주차면수는 샘플 데이터를 사용합니다.

향후 실제 서비스에서는 전국 공영주차장과 실시간 공공데이터 API를 연결하여 전국 단위 서비스로 확장할 수 있도록 설계합니다.

---

# 3. 핵심 기능

## 3-1 Home

서비스 첫 화면입니다.

필수 기능

- 현재 위치 가져오기
- 카카오맵 표시
- 현재 위치 중심 이동
- 검색창
- 서울 공영주차장 약 20~30개 Marker 표시
- 현재 위치 버튼(Floating Button)

---

## 3-2 지도

카카오맵 SDK를 사용합니다.

지도에는 서울시 주요 공영주차장 약 20~30개를 표시합니다.

Marker는 혼잡도에 따라 색상이 달라집니다.

혼잡도 기준

🟢 여유

🟡 보통

🔴 혼잡

⚫ 만차

Marker는 확대 및 축소 시 자연스럽게 표시되어야 합니다.

Marker 클릭 시 선택 애니메이션을 적용합니다.

---

## 3-3 Bottom Sheet

Marker를 클릭하면 화면 하단에서 Bottom Sheet가 올라옵니다.

Bottom Sheet에는 아래 정보를 표시합니다.

- 주차장명
- 주소
- 운영시간
- 요금
- 총 주차면수
- 현재 가능 주차면수
- 혼잡도
- 마지막 업데이트 시간
- 길찾기 버튼

Bottom Sheet는 모바일 UI처럼 부드럽게 올라오는 애니메이션을 적용합니다.

---

## 3-4 검색

사용자는 주차장 이름을 검색할 수 있습니다.

검색 결과는 자동완성을 지원합니다.

검색 시 해당 Marker로 이동합니다.

---

## 3-5 현재 위치

현재 위치 버튼을 누르면

사용자의 현재 위치로 다시 이동합니다.

---

## 3-6 길찾기

Bottom Sheet의 길찾기 버튼을 누르면

카카오맵 길찾기 페이지로 이동합니다.

---

# 4. 데이터 구조

이번 MVP에서는 실제 존재하는 서울 공영주차장을 사용합니다.

- 지역
    - 강남구
    - 종로구
    - 중구
    - 성동구

약 20~30개의 공영주차장을 사용합니다.

- 실제 사용하는 데이터
    - 주차장명
    - 주소
    - 위도
    - 경도
- 샘플 데이터
    - 총 주차면수
    - 현재 가능면수
    - 혼잡도
    - 업데이트 시간
- 혼잡도는 아래 기준으로 계산합니다.
    - 여유 (60%이상)
    - 보통 (30~59%)
    - 혼잡 (1~29%)
    - 만차 (0면)

# 8. MVP 범위

이번 메이커톤에서는

서울 공영주차장 약 20~30개를 대상으로 MVP를 구현합니다.

실시간 공공데이터 API는 연동하지 않습니다.

대신 API를 쉽게 연결할 수 있는 구조로 설계합니다.

현재 가능 주차면수와 혼잡도는 샘플 데이터를 사용합니다.

실제 서비스에서는 공공데이터포털 및 지자체 실시간 주차 API를 연결하여 전국 단위 서비스로 확장할 수 있도록 설계합니다.

# 9. 개발 요청사항

Claude는 예제 수준이 아닌 실제 서비스 수준으로 구현합니다.

반응형 UI를 적용합니다.

모바일 우선으로 구현합니다.

컴포넌트를 분리하여 유지보수가 쉽도록 작성합니다.

Loading UI를 제공합니다.

Error UI를 제공합니다.

Empty State를 제공합니다.

Skeleton UI를 제공합니다.

검색 기능을 제공합니다.

현재 위치 버튼을 제공합니다.

Bottom Sheet 애니메이션을 제공합니다.

향후 실시간 API를 쉽게 연결할 수 있도록 데이터 구조를 설계합니다.

실제 서비스 수준의 깔끔하고 현대적인 UI를 구현합니다.

# 10. 데이터 관리 규칙

주차장 데이터는 반드시 `/data/parking_lots.json` 파일 하나만 읽도록 구현합니다.

주차장 데이터는 코드 내부에 하드코딩하지 않습니다.

모든 Marker와 주차장 정보는 `parking_lots.json`에서 불러옵니다.

향후 실제 공공데이터 API를 연동할 때도 `parking_lots.json`과 동일한 데이터 구조를 유지하여 쉽게 교체할 수 있도록 설계합니다.
